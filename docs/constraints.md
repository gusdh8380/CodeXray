# Constraints & Risks

## Principles
- 도구는 판단 **근거**이지 판단 자체가 아니다. 최종 결정은 사람이.
- 민감 코드 보호: 로컬 실행 모드를 기본으로, AI 평가는 **사용자 opt-in**.

## Stack
- 미정. vibe-coding 진행 중 결정 (사용자는 스택 선택에 관여하지 않음).

## Top risk — AI 평가 부정확 → 잘못된 의사결정
가장 두려운 실패. 이를 막기 위해 모든 AI 평가는:
1. **근거 코드 라인 인용 필수** — 점수만 던지지 않음
2. **신뢰도·한계 명시** — "이 평가는 X를 보지 못함"
3. **사용자가 거절·재평가 가능** — 최종 판단권은 사용자

## First validation target
1차 MVP의 검증 입력은 **사용자 본인이 만든 작은 게임 프로젝트**.
→ "이 게임 프로젝트에서 의미 있는 분석·평가가 나오는가" 가 MVP 통과 기준선.
