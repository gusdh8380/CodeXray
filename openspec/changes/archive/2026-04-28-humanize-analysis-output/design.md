## Context

현재 분석 탭들이 숫자와 전문 용어를 맥락 없이 노출한다. render.py의 각 render 함수에서 표현 방식을 개선하면 된다. 별도 라이브러리나 외부 의존성 없이 Python 헬퍼 함수로 처리 가능하다.

## Goals / Non-Goals

**Goals:**
- 주요 용어(fan-in, fan-out, coupling, LoC, SCC, hotspot 등)에 괄호 설명 추가
- LoC 수치를 규모 레이블로 변환 (예: ~1K, ~8.8K → "소규모", "중규모", "대규모")
- coupling 수치를 위험도로 해석 (낮음/보통/높음/매우높음)
- Quality 등급(A~F)에 한 줄 해석 추가
- Hotspot 카테고리를 한국어 + 설명으로 표시
- Overview의 숫자 카드에 맥락 설명 추가

**Non-Goals:**
- UI 구조 변경 (레이아웃, 탭 등)
- 새로운 분석 지표 추가
- 툴팁 팝업 등 JS 인터랙션 (괄호 설명으로 충분)

## Decisions

### 1. 헬퍼 함수 위치
`render.py` 내부에 `_humanize_*` 함수들을 추가. 별도 모듈 불필요.

### 2. LoC 규모 레이블
- < 1,000: 소규모
- 1,000 ~ 10,000: 중규모
- 10,000 ~ 50,000: 대규모
- > 50,000: 초대형

### 3. Coupling 위험도
- 0~5: 낮음
- 6~15: 보통
- 16~30: 높음
- > 30: 매우 높음

### 4. 용어 설명 형식
`fan-in (이 파일에 의존하는 파일 수)` — 괄호로 짧게. 과도한 설명은 오히려 방해.

### 5. Hotspot 카테고리 한국어화
- hotspot → 위험 (변경 잦고 결합도 높음)
- active_stable → 안정 활성 (변경 잦지만 결합도 낮음)
- neglected_complex → 방치 복잡 (변경 적지만 결합도 높음)
- stable → 안정

## Risks / Trade-offs

- 텍스트가 길어지면 테이블 레이아웃이 깨질 수 있음 → 괄호 설명은 짧게 유지
- 임계값이 임의적 — 프로젝트 특성에 따라 다를 수 있음, 단순 가이드라인으로 취급
